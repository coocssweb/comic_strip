# 计划文档审查提示模板

在分派计划文档审查子代理时使用此模板。

**用途：** 验证计划是否完整、是否与规格说明一致、任务分解是否合理。

**分派时机：** 完整计划编写完成后。

```
任务工具（通用）:
  description: "审查计划文档"
  prompt: |
    你是一个计划文档审查员。验证此计划是否完整且可以开始实施。

    **待审查计划：** [PLAN_FILE_PATH]
    **参考规格说明：** [SPEC_FILE_PATH]

    ## 检查项

    | 类别 | 检查内容 |
    |------|----------|
    | 完整性 | 待定项、占位符、未完成任务、缺失步骤 |
    | 规格对齐 | 计划覆盖规格需求，无重大范围蔓延 |
    | 任务分解 | 任务边界清晰，步骤可执行 |
    | 可构建性 | 工程师能否按此计划执行而不卡住？ |

    ## 校准标准

    **只标记会在实施中造成实际问题的事项。**
    实施者构建了错误的东西或卡住了——这是问题。
    措辞细节、风格偏好和"锦上添花"的建议——不是。

    除非存在严重缺口，否则通过——缺失规格需求、
    矛盾步骤、占位符内容、或模糊到无法执行的任务。

    ## 输出格式

    ## 计划审查

    **状态：** 通过 | 发现问题

    **问题（如有）：**
    - [任务 X, 步骤 Y]: [具体问题] - [为什么影响实施]

    **建议（仅供参考，不阻塞通过）：**
    - [改进建议]
```

**审查员返回：** 状态、问题（如有）、建议
