# 规格文档审阅提示模板

规格文档编写完成后，使用此模板进行审阅。

**用途：** 验证规格是否完整、一致，可以进入实现计划阶段。

**触发时机：** 规格文档写入 `docs/specs/` 之后

## 审阅检查项

| 类别 | 检查内容 |
|------|---------|
| 完整性 | TODO、占位符、"TBD"、未完成的章节 |
| 一致性 | 内部矛盾、冲突的需求 |
| 清晰度 | 需求是否模糊到可能导致实现偏差 |
| 范围 | 是否足够聚焦，可以用一份计划覆盖 — 而非涵盖多个独立子系统 |
| YAGNI | 未被要求的功能、过度设计 |

## 校准标准

**只标记会在实现计划阶段造成实际问题的事项。**

缺失的章节、矛盾的描述、或模糊到可以被两种方式理解的需求 — 这些是问题。措辞微调、风格偏好、"某些章节不如其他章节详细" — 这些不是。

除非存在会导致计划出错的严重缺陷，否则通过审阅。

## 输出格式

```
## 规格审阅

**状态：** 通过 | 发现问题

**问题（如有）：**
- [章节 X]：[具体问题] - [为什么影响计划]

**建议（仅供参考，不阻塞通过）：**
- [改进建议]
```
