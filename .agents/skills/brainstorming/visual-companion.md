# 可视化辅助指南

头脑风暴过程中，基于浏览器的可视化辅助工具，用于展示模型图、图表和选项。

## 何时使用

逐个问题决策，而非整个会话统一决定。判断标准：**用户看到它是否比读到它更容易理解？**

**用浏览器** 展示本身就是视觉的内容：

- **UI 模型图** — 线框图、布局、导航结构、组件设计
- **架构图** — 系统组件、数据流、关系图
- **并排视觉对比** — 对比两种布局、两种配色、两种设计方向
- **设计细节** — 当问题涉及外观、间距、视觉层次
- **空间关系** — 状态机、流程图、实体关系图

**用终端** 处理文本或表格内容：

- **需求和范围问题** — "X 是什么意思？"、"哪些功能在范围内？"
- **概念性 A/B/C 选择** — 用文字描述的方案选择
- **权衡列表** — 优缺点、对比表
- **技术决策** — API 设计、数据建模、架构方案选择
- **澄清问题** — 任何答案是文字而非视觉偏好的问题

关于 UI 话题的问题不一定是视觉问题。"你想要什么样的向导？"是概念问题 — 用终端。"这几种向导布局哪个感觉对？"是视觉问题 — 用浏览器。

## 工作原理

在 Kiro 环境中，可视化辅助可以通过以下方式实现：

1. **Figma MCP 工具** — 如果有 Figma 设计稿，直接通过 Figma Power 获取和展示设计
2. **Playwright 浏览器工具** — 可以在浏览器中渲染 HTML 模型图供用户查看
3. **截图** — 对已有页面截图进行对比讨论

## 使用流程

1. **判断是否需要视觉展示** — 对每个问题单独判断
2. **选择展示方式** — 根据内容类型选择合适的工具
3. **展示并说明** — 告诉用户在看什么，给出简要文字描述
4. **收集反馈** — 询问用户的看法和选择
5. **迭代或推进** — 如果反馈需要修改当前展示，更新后重新展示；确认后进入下一个问题

## 内容类型与展示建议

### 选项展示（A/B/C 选择）

适合展示 2-4 个视觉方案供用户选择，每个方案包含：
- 标题
- 简要描述
- 视觉预览（如适用）

### 模型图展示

适合展示：
- 页面布局线框图
- 组件结构示意
- 导航流程

### 并排对比

适合展示：
- 两种布局方案的对比
- 修改前后的对比
- 不同设计方向的对比

## 设计建议

- **展示精度与问题匹配** — 布局问题用线框图，视觉细节问题用高保真
- **每页说明问题** — "哪种布局更专业？"而不是只说"选一个"
- **确认后再推进** — 如果反馈需要修改当前展示，先更新再继续
- **每屏最多 2-4 个选项**
- **保持模型图简洁** — 聚焦布局和结构，不追求像素级完美
