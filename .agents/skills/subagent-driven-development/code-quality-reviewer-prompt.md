# 代码质量审查者提示模板

派发代码质量审查者子代理时使用此模板。

目的：验证实现构建良好（整洁、已测试、可维护）

仅在规范合规性审查通过后派发。

```
Task 工具（通用）：
  使用 requesting-code-review/code-reviewer.md 中的模板

  DESCRIPTION: [任务摘要，来自实施者的报告]
  PLAN_OR_REQUIREMENTS: 计划文件中的任务 N
  BASE_SHA: [任务前的提交]
  HEAD_SHA: [当前提交]

```

除了标准代码质量问题外，审查者应检查：

- 每个文件是否有一个清晰的职责和明确定义的接口？
- 单元是否分解为可以独立理解和测试？
- 实现是否遵循计划中的文件结构？
- 此实现是否创建了已经很大的新文件，或显著增加了现有文件的大小？（不要标记预先存在的文件大小 —— 关注此更改贡献的内容。）

代码审查者返回：优点、问题（关键/重要/次要）、评估
