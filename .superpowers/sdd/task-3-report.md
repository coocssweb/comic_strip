# 任务 3 实施报告

## 实施内容

- 新增 `User` Mongoose 模型：名称、邮箱、年龄及创建/更新时间。
- 新增无鉴权用户 CRUD 控制器：创建、列表、详情、更新、删除。
- 新增 Joi 请求校验：创建时名称和邮箱必填；更新至少包含一个可更新字段；用户 ID 限制为 24 位十六进制字符串。
- 新增根路由：暴露 `GET /health` 与 `/api/users` 下的五个用户接口。

## 错误与安全处理

- 用户不存在统一抛出 `40401`，可见提示为“用户不存在”。
- 请求参数错误交由既有 `validate` 中间件统一返回 `40001`。
- 数据库重复邮箱、非法 ID 与未知异常继续由既有错误中间件脱敏处理，未暴露原始异常或敏感数据。
