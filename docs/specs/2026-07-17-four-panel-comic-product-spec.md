# 四格漫画小程序产品规格

本规格综合当前已确认的产品决策、领域词汇与 ADR。它是实现前的产品输入；详细接口以配套 API 契约为准。

## 问题陈述

用户当前只能浏览四格漫画原型，无法在微信小程序中读取真实内容、与内容互动或通过专题和月度热度榜发现作品。管理员也缺少统一且受保护的后台来维护漫画系列、漫画单话、主题标签、运营专题、评论和图片。

## 解决方案

在现有 Koa + MongoDB 架构上建设原生微信小程序、管理后台与共享 API。管理员维护运营内容并通过腾讯云 COS 上传图片；读者可匿名阅读已发布漫画单话，登录后点赞、收藏、评论和评论点赞，并通过微信原生能力分享单话。首页推荐流、阅读器、漫画系列、运营专题、月度热度榜与“我的”共同组成读者闭环。

## 用户故事

1. 作为匿名读者，我想浏览首页推荐流，以便无需登录即可发现新发布的漫画单话。
2. 作为读者，我想在首页横向预览同一单话的四个画格，以便快速判断是否想阅读。
3. 作为读者，我想在阅读器中横向切换四个画格，以便连贯阅读一篇漫画单话。
4. 作为读者，我想在阅读器中纵向切换相邻推荐单话，以便连续阅读首页推荐流。
5. 作为读者，我想从单话进入漫画系列页，以便阅读该系列所有已发布单话。
6. 作为读者，我想浏览运营专题，以便按人工编排的主题发现跨系列内容。
7. 作为读者，我想查看月度热度榜，以便发现当月互动更活跃的漫画系列。
8. 作为读者，我想在未登录状态下阅读公开内容，以便先体验产品。
9. 作为读者，我想使用微信登录，以便获得点赞、收藏和评论权限。
10. 作为读者，我想在授权后展示昵称与头像，以便在评论中获得可识别的身份。
11. 作为登录读者，我想对漫画单话点赞或取消点赞，以便表达认可。
12. 作为登录读者，我想收藏或取消收藏漫画单话，以便在“我的收藏”中稍后阅读。
13. 作为登录读者，我想发表一级纯文本评论，以便表达对单话的反馈。
14. 作为登录读者，我想点赞或取消点赞一条评论，以便认可其他读者的观点。
15. 作为评论作者，我想删除自己的评论，以便撤回不希望展示的内容。
16. 作为读者，我想在“我的”中查看收藏、单话点赞和未删除评论，以便回顾互动记录。
17. 作为读者，我想分享当前漫画单话给微信好友或群，以便邀请他人阅读。
18. 作为接收分享的读者，我想直接打开对应的漫画单话，以便快速查看分享内容。
19. 作为管理员，我想登录独立管理后台，以便安全维护所有运营内容。
20. 作为管理员，我想管理主题标签并禁止删除仍被单话引用的标签，以便保持内容归类完整。
21. 作为管理员，我想创建漫画系列、维护作者署名和创建固定四画格的漫画单话，以便组织运营内容。
22. 作为管理员，我想发布或下架漫画单话，并在编辑已发布单话前先下架，以便避免读者看到编辑中的内容。
23. 作为管理员，我想上传专题封面、维护运营专题及其排序，以便运营跨系列内容集合。
24. 作为管理员，我想通过短时预签名将画格和专题封面直传腾讯云 COS，以便避免后端中转图片二进制。
25. 作为管理员，我想查看并删除任意评论，以便执行最小限度的人工处置。
26. 作为运营者，我想让月度热度榜以单话点赞、评论创建、评论点赞和分享次数计算热度，以便采用简单透明的排行规则。
27. 作为读者，我想在单话下架后无法从公开入口打开它，以便只看到当前可用内容。
28. 作为管理员，我想在重新发布下架单话后恢复其历史互动，以便纠错时不丢失数据。

## 实现决策

- 范围覆盖 backend、mini-program、admin-web 和部署配置；三端共用版本化 REST API，成功与失败均使用稳定业务 `code` 和中文用户提示。
- 核心模型包括漫画系列、漫画单话、画格、主题标签、运营专题、读者账户、评论、点赞、收藏、评论点赞和分享记录。
- 漫画单话固定四个有序画格并单选一个管理员维护的主题标签；系列缩略图复用最新已发布单话的首画格，专题封面单独上传。
- 单话状态固定为 `draft → published ↔ unpublished`。已发布单话不可直接编辑，必须先下架；下架后不出现在读者端、系列页、专题页或月榜，但保留互动并在重新发布后恢复。
- 首页推荐流只读取已发布单话并按发布时间倒序分页；阅读器横向切画格、纵向切首页推荐流单话。
- 读者通过微信登录凭证由后端兑换唯一标识，并使用独立读者会话；授权昵称与头像仅在登录时同步，不提供资料编辑。
- 管理后台只存在一个管理员，账号和密码哈希来自部署环境变量；管理员会话与读者会话严格隔离。
- 单话点赞、收藏、评论点赞按读者与目标建立唯一关系且均可取消；“我的点赞”只收录单话点赞。
- 评论为 1–200 字符的一级纯文本，允许表情和换行，不允许图片、链接、@、回复或楼中楼；评论按创建时间倒序分页，逻辑删除后不在任何读者端列表显示。
- 每次发起单话分享都计数，不做去重。月度热度榜按系列聚合当月公开单话的点赞、评论创建、评论点赞和分享次数，各计一分；同分按分享次数排序。
- 漫画图片使用腾讯云 COS：公开 HTTPS 读取，后端签发短时预签名，管理后台浏览器直传，后端只保存 COS URL。仅允许 JPEG、PNG、WebP，单张最大 5 MB。
- 主题标签支持新增、改名、排序；被任一单话使用时不能删除。运营专题由管理员手动排序，且仅包含已发布单话。
- 现有原型提供阅读器横向四画格、纵向切换单话、固定互动区及三栏信息架构的交互依据；不直接复用其演示状态或模拟数据。
- 具体数据约束、API 路径、参数、响应字段、错误语义与权限，遵循 `2026-07-17-four-panel-comic-api.md`；COS 和管理员认证遵循 ADR-0001、ADR-0002。

## 测试决策

- 好的测试只验证外部可观察行为，不绑定组件内部状态或模型实现细节。
- 最高验证切面是本地真实闭环：管理员上传并发布内容，读者端通过真实 API 阅读并互动，管理员下架后读者不可访问，重新发布后互动恢复，系列、专题和月榜均能读取。
- 后端测试覆盖微信登录与会话隔离、四画格校验、状态流转、公开内容过滤、互动唯一性与取消、评论删除权限、标签删除限制、月榜公式、COS URL 白名单与预签名限制。
- 小程序验证匿名阅读、登录互动、评论分页、我的数据、分享计数以及横向画格和纵向单话切换。
- 管理后台验证管理员登录、标签管理、四图直传和排序、草稿保存、发布下架、专题排序和评论删除。
- backend、mini-program、admin-web 的测试分别放在各自 `tests` 目录；真实联调不能被构建、单测或静态审查替代。

## 范围外

- 个性化推荐、搜索、读者端筛选、评分、打卡、奖励、海报和裂变。
- 作者账号、作者主页、投稿与创作端、关注订阅、收藏夹和系列收藏。
- 评论回复、楼中楼、图片评论、链接评论、@提及、审核、举报和违规处置。
- 多管理员账号、角色权限管理、通知与正式环境数据操作。

## 补充说明

- 无审核/举报和分享不去重是已确认风险；上线期间由管理员人工关注评论内容和异常热度。
- 部署前必须配置微信小程序凭据、管理员账号密码哈希、会话密钥、COS 桶与地域、公开访问域名、上传有效期和 COS 访问凭据；任何密钥不得进入前端、日志或仓库。
- 严格执行顺序为：用户确认规格与契约 → 后端实现与验证 → 小程序和管理后台并行实现 → 真实联调 → 跨端审查。
