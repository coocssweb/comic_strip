# 四格漫画小程序与管理后台设计

日期：2026-07-17  
状态：已确认，作为后续实现的冻结设计输入  
风险等级：高

## 1. 目标与边界

在既有 Koa + MongoDB 后端上新增四格漫画内容与互动能力。原生微信小程序和管理后台共用同一组后端 API；图片存腾讯云 COS，后端仅保存符合已配置 COS 公网域名的 HTTPS URL。

本期只覆盖以下闭环：管理员管理标签、创建系列、创建固定四格的单话、发布和下架；读者浏览、点赞、收藏、评论、评论点赞和分享；管理员运营系列月榜与跨系列专题。

明确不做：个性化推荐、搜索、筛选、评分、打卡、关注、作者端、海报、奖励、裂变、内容审核、举报、评论回复、图片或链接评论、@ 提及。

## 2. 影响范围

| 范围 | 影响 |
| --- | --- |
| `backend` | 新增读者登录、读者会话、内容/互动/榜单/后台管理/COS 预签名 API，新增 MongoDB 集合、索引与聚合查询。 |
| `mini-program` | 新增首页、阅读器、系列页、专题页、发现页、我的页及互动操作；接入微信登录和原生单话分享。 |
| `admin-web` | 新增唯一管理员登录、系列/单话/专题管理、COS 浏览器直传、发布下架和评论删除。 |
| 部署配置 | 新增微信与 COS 环境变量；管理员账号和密码哈希继续只放环境变量。 |

## 3. 领域模型与数据约束

### 3.1 漫画系列 `comic_series`

字段：`id`、`name`、`summary`、`authorByline`、`createdAt`、`updatedAt`。系列缩略图始终复用其最新已发布单话的首画格，不维护独立封面字段。

- `name` 必填，系列页按系列聚合其已发布单话。
- 系列只能在无任何单话时删除，避免删除后读者互动失去归属。

### 3.2 漫画单话 `comic_episodes`

字段：`id`、`seriesId`、`title`、`summary`（可空）、`themeTagId`、`panels`、`status`、`publishedAt`、`createdAt`、`updatedAt`。

`panels` 永远是长度为 4 的有序数组，每项为 `{ position: 1|2|3|4, imageUrl, altText? }`；`position=1` 是全部列表和分享卡片使用的缩略图。`themeTagId` 关联一个不可为空的管理员维护标签；一话只可选择一个标签。

状态固定为：

```text
草稿 draft -> 已发布 published <-> 已下架 unpublished
```

- 仅完整的四格内容可以发布；首次发布写入 `publishedAt`。
- `published` 不允许编辑任何内容字段；必须先下架，变为 `unpublished` 后才能编辑，再次发布。
- 下架单话不得在读者端的任何列表、详情、分享落地页、系列页、专题页和榜单中出现，也不得直接打开；其点赞、收藏、评论、评论点赞和分享统计保留，重新发布后恢复。
- 仅 `draft` 单话可删除；已发布或已下架单话必须通过下架保留数据，不提供物理删除。

### 3.3 主题标签 `comic_tags`

字段：`id`、`name`、`sortOrder`、`createdAt`、`updatedAt`。标签仅用于单话展示，不提供读者端筛选。管理员可新增、改名和调整排序；被任一单话引用的标签不得删除，必须先修改关联单话的标签。

### 3.4 读者与会话 `readers`、`reader_sessions`

读者字段：`id`、`wechatOpenId`、`displayName`、`avatarUrl`、`createdAt`、`lastLoginAt`。微信登录凭证仅提交给后端，由后端用环境变量中的 `WECHAT_APP_ID`、`WECHAT_APP_SECRET` 换取微信用户唯一标识并创建或复用读者账户；昵称与头像仅在用户授权登录时同步，不提供个人资料编辑功能。小程序和浏览器均不得接触 AppSecret。

系统会话以服务端可验证的读者会话令牌表示，接口通过 `Authorization: Bearer <token>` 传递；令牌不可作为日志字段输出。

### 3.4 互动数据

| 集合 | 核心字段与唯一约束 |
| --- | --- |
| `episode_likes` | `readerId`、`episodeId`、`createdAt`；唯一索引 `(readerId, episodeId)`。 |
| `episode_favorites` | `readerId`、`episodeId`、`createdAt`；唯一索引 `(readerId, episodeId)`。 |
| `comments` | `readerId`、`episodeId`、`content`、`createdAt`、`deletedAt`、`deletedByRole`、`deletedById`；逻辑删除留痕。 |
| `comment_likes` | `readerId`、`commentId`、`createdAt`；唯一索引 `(readerId, commentId)`。 |
| `episode_shares` | `episodeId`、`readerId?`、`createdAt`；每次分享请求各插入一条，无去重。 |

评论只有一级纯文本，长度按 Unicode 字符计为 1 至 200，允许 emoji 和换行；拒绝图片、链接、`@` 回复语义。删除后的评论不在读者评论列表、单话详情或“我的评论”出现，但保留后台审计字段。评论点赞只公开计数，且不出现在“我的点赞”。

### 3.5 专题 `editorial_topics`

专题用于管理员手动编排的跨系列内容集合，字段为：`id`、`title`、`summary`、`coverImageUrl`、`episodeIds`、`createdAt`、`updatedAt`。`episodeIds` 的数组顺序就是展示顺序；仅允许包含已发布单话。删除专题只删除专题与其编排关系，不删除任何单话或互动。

### 3.6 索引和统计

- `comic_episodes`：`(status, publishedAt desc)`、`(seriesId, status, publishedAt desc)`。
- `comments`：`(episodeId, deletedAt, createdAt desc)`、`(readerId, deletedAt, createdAt desc)`。
- 点赞、收藏、评论点赞均以复合唯一索引保证幂等约束。
- 月榜按系列和自然月（部署时区固定 `Asia/Shanghai`）聚合：该系列已发布单话在当月产生的单话点赞数 + 评论创建数 + 评论点赞数 + 分享次数；每项一分。同分时分享次数高者优先，再以系列 `id` 升序保证稳定顺序。下架单话在统计窗口内的历史互动不再计入公开月榜。

## 4. 页面与交互

### 4.1 小程序

- 首页：只展示已发布单话，按 `publishedAt` 倒序游标分页；首格缩略图、标题、系列、主题标签和互动计数。
- 阅读器：当前单话横向滑动四个画格；纵向切换首页推荐流中的相邻已发布单话。直接打开下架或不存在单话时显示“内容暂不可用”，不泄露下架原因。
- 系列页：展示系列信息和该系列已发布单话。
- 专题页：展示管理员手动排序的跨系列已发布单话；失效或下架项由后端过滤，前端不补位排序。
- 发现页：按系列展示自然月榜；专题入口和内容来自管理员排序。
- 我的页：未登录时引导微信登录；登录后展示我的收藏、我的单话点赞、我的评论，均倒序分页。已删除评论不展示。
- 分享：使用微信原生分享单话路径；在每次发起分享时调用计数 API。无海报、奖励或裂变。

### 4.2 管理后台

- 管理员登录后可管理标签、系列、单话和专题。
- 单话编辑器强制维护四张图片及排序；单张画格和专题封面仅允许 JPEG、PNG、WebP，且不超过 5 MB。后台先请求后端短时预签名，浏览器以 `PUT` 直传 COS，提交单话时只提交返回的公开 HTTPS URL。
- 已发布单话编辑入口必须先执行下架；发布、下架有明确状态提示。
- 可删除任意评论，删除为逻辑删除并记录操作者。

## 5. 权限与错误边界

| 角色 | 能力 |
| --- | --- |
| 匿名读者 | 读取全部读者端公开内容；可触发分享计数。 |
| 已登录读者 | 匿名能力之外，可点赞、收藏、评论、评论点赞、查看个人数据、删除自己的评论。 |
| 唯一管理员 | 管理后台的全部内容、状态、COS 预签名及任意评论删除。 |

管理员身份使用环境变量中的账号和密码哈希校验，后台会话令牌与读者会话隔离。所有用户可见错误均为中文；鉴权、校验、状态冲突、资源不存在和下架资源均以稳定业务 `code` 返回，前端不得只依赖 HTTP 状态。

## 6. 验收与测试

后端至少覆盖：微信登录建档与会话、单话四格校验、状态机和已发布编辑拦截、公开内容过滤、互动唯一性与取消、评论内容/逻辑删除、角色删除权限、月榜公式和同分规则、专题下架过滤、COS 预签名与 URL 白名单。

小程序至少覆盖：匿名阅读、登录后互动、评论分页、我的数据、已下架内容不可打开、原生分享计数调用、横四格和纵单话切换。

管理后台至少覆盖：管理员登录、四图直传和排序、草稿保存、发布/下架、已发布编辑拦截、专题排序、删除任意评论。

高风险真实联调必须启动本地后端及两个前端，使用本地或脱敏数据完成“后台上传并发布 → 小程序读取和互动 → 下架不可见 → 重发恢复互动 → 月榜与专题读取”的闭环，并记录请求与关键响应；构建或单测不能替代该验证。

## 7. 部署配置

| 配置 | 用途 |
| --- | --- |
| `WECHAT_APP_ID` | 后端向微信交换登录凭证。 |
| `WECHAT_APP_SECRET` | 后端专用微信密钥，严禁下发客户端或写入日志。 |
| `ADMIN_USERNAME` | 唯一管理员账号。 |
| `ADMIN_PASSWORD_HASH` | 唯一管理员密码哈希。 |
| `SESSION_SECRET` | 读者与管理员会话签发/验证密钥。 |
| `COS_BUCKET`、`COS_REGION` | 腾讯云 COS 存储位置。 |
| `COS_PUBLIC_BASE_URL` | 唯一允许持久化的 COS 公网 HTTPS URL 前缀。 |
| `COS_UPLOAD_EXPIRES_SECONDS` | 管理端预签名有效期，取短时值。 |
| `COS_ACCESS_KEY_ID`、`COS_SECRET_ACCESS_KEY` | 后端生成 COS 预签名，严禁下发前端。 |

继续遵循既有 ADR-0001：公开读、后端签发短时预签名、管理后台直传；继续遵循 ADR-0002：单管理员、环境变量账号/密码哈希和后台会话。

## 8. 已知风险

1. 无内容审核和举报，漫画与评论可能被滥用；上线运营必须由管理员人工处置并保留删除审计。
2. 分享按每次发起直接计数且无去重，可能被脚本或重复操作抬高热度；这是已确认业务规则，月榜不应被解释为唯一用户热度。
3. 公开读 COS URL 依赖上传前后端白名单校验和短时预签名；不得接受任意外部 URL。
4. 微信登录、会话和管理员身份均涉及认证边界，日志、错误和监控中不得记录凭证、会话令牌、密码哈希或 AppSecret。

## 9. 严格执行顺序

1. Main Agent 将本设计和 API 契约向用户确认。
2. 确认后先由 Backend Agent 实现、测试并同步最终契约。
3. Backend 完成和验证通过后，由 Mini Program Agent 与 Admin Web Agent 并行实现各自端。
4. 两端完成后，由 Integration Verification Agent 执行真实联调闭环。
5. 联调通过后，由 Review Agent 进行跨端审查；发现问题由 Main Agent 分派至责任端修复并复验。
