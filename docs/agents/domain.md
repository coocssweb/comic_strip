# 领域文档

工程技能探索代码库时，按本文约定读取领域文档。

## 探索前读取

- 读取仓库根目录的 `CONTEXT.md`。
- 如果根目录存在 `CONTEXT-MAP.md`，则按映射读取与当前任务有关的上下文文档。
- 读取 `docs/adr/` 中与当前工作区域相关的 ADR。

如果上述文件或目录尚不存在，继续执行任务，不需要提前提示或创建。领域词汇与架构决策在实际形成后，由领域建模相关技能按需创建。

## 文件布局

本仓库采用单上下文布局：

```text
/
├── CONTEXT.md
└── docs/
    └── adr/
```

## 使用领域词汇

议题标题、重构方案、故障假设和测试名称应使用 `CONTEXT.md` 定义的领域术语，避免使用词汇表明确排除的同义词。

如果所需概念尚未记录，应先判断是否使用了项目之外的语言；确属领域词汇缺口时，将其交给领域建模流程补充。

## ADR 冲突处理

如果方案与现有 ADR 冲突，必须明确指出冲突及重新讨论该决策的原因，不得静默覆盖。
