# 项目规则

本项目所有对话、分析、代码修改和文档生成必须遵守以下规则。 

## 安全铁律

- 严格禁止主动连接正式环境数据库、操作正式环境数据库、查看正式环境数据；即使为了排查问题，也必须拒绝此类主动行为，并改用本地、开发、测试或脱敏数据方案。

## 测试铁律

- TDD 或其它测试相关改动的测试文件必须集中放在`backend/tests` 目录下，严格禁止直接放在业务代码目录或业务文件旁边。

## 目录与架构规范

- `src/models/`: MongoDB Schema 与静态方法定义。命名以 `.model.js` 结尾。
- `src/controllers/`: 请求参数校验、调用 Service/Model 以及向客户端返回响应。命名以 `.controller.js` 结尾。
- `src/routes/`: 路由声明和中间件绑定。命名以 `.route.js` 结尾。
- `src/utils/`: 纯函数或与业务无关的通用工具。

## 错误处理模式

- 控制器中应该使用类似 `ctx.fail(400, '错误信息')` 或通过统一中间件抛出结构化的 API 异常。
- 不要在 Model 层直接向客户端输出响应。
- 使用事务 (Session) 时，如果涉及多个跨表写操作，确保在异常捕获时 `session.abortTransaction()`。
