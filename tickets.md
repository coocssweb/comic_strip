# Tickets: 四格漫画小程序第一期

基于 `docs/specs/2026-07-17-four-panel-comic-product-spec.md` 的可独立执行任务清单。按依赖关系推进：所有阻塞项完成后，任务才进入可领取前沿。

## 后端：认证与 COS 上传基础

**What to build:** 管理员和读者获得彼此隔离的登录会话；管理员可获取短时 COS 预签名地址并从浏览器直传符合限制的漫画图片或专题封面。

**Blocked by:** None — can start immediately.

- [ ] 微信登录凭证可在后端兑换读者账户与读者会话，授权昵称和头像仅在登录时同步。
- [ ] 单管理员可通过环境变量账号与密码哈希登录，且管理员会话不能调用读者接口，反之亦然。
- [ ] 管理员可取得仅允许 JPEG、PNG、WebP 且单张不超过 5 MB 的 COS 短时预签名；前端只能持久化允许的 COS 公开 HTTPS URL。

## 后端：内容与发现 API

**What to build:** 管理员可维护主题标签、漫画系列、固定四画格的漫画单话和运营专题；读者端能够读取首页推荐流、系列、专题与月度热度榜。

**Blocked by:** 后端：认证与 COS 上传基础。

- [ ] 主题标签支持新增、改名、排序；被漫画单话引用时不可删除。
- [ ] 漫画单话仅能在具有完整四个画格时发布，并严格执行草稿、已发布、已下架的状态流转与已发布内容先下架后编辑的规则。
- [ ] 首页推荐流、漫画系列、运营专题和月度热度榜只返回当前已发布内容；系列缩略图复用首画格，专题保持管理员排序和独立封面。
- [ ] 月度热度榜按已确认的互动公式与同分规则排序，已下架单话不计入公开榜单。

## 后端：读者互动 API

**What to build:** 登录读者可以对已发布漫画单话完成点赞、收藏、评论、评论点赞和分享计数，并在“我的”中读取自己的有效互动。

**Blocked by:** 后端：认证与 COS 上传基础；后端：内容与发现 API。

- [ ] 单话点赞、收藏与评论点赞对同一读者和目标保持唯一、可取消并返回公开计数。
- [ ] 评论仅支持 1–200 字符一级纯文本，按创建时间倒序分页；作者可删除自己的评论，删除后读者端不再展示。
- [ ] 每次单话分享均记数；我的收藏、单话点赞和未删除评论按创建时间倒序返回。
- [ ] 下架单话不可互动或公开访问，但重新发布后可恢复历史互动。

## 小程序：公开阅读与发现

**What to build:** 匿名读者能够通过真实 API 浏览首页推荐流、阅读器、漫画系列、运营专题和月度热度榜，并通过分享链接直达公开单话。

**Blocked by:** 后端：认证与 COS 上传基础；后端：内容与发现 API；后端：读者互动 API。

- [ ] 首页按发布时间倒序分页展示已发布漫画单话及真实互动计数。
- [ ] 阅读器横向切换四个画格、纵向切换首页推荐流相邻单话，并在不可用内容时展示中文不可用提示。
- [ ] 系列页、专题页和月度热度榜展示后端筛选后的公开内容；微信分享进入目标单话。

## 小程序：登录互动与“我的”

**What to build:** 登录读者能够在小程序中完成互动，并从“我的”管理其收藏、单话点赞与未删除评论。

**Blocked by:** 后端：认证与 COS 上传基础；后端：内容与发现 API；后端：读者互动 API；小程序：公开阅读与发现。

- [ ] 未登录读者可阅读，发起写操作时被引导微信登录；登录后显示授权昵称和头像。
- [ ] 单话点赞、收藏、评论、评论点赞和分享计数调用真实 API，并正确刷新用户可见计数与状态。
- [ ] 我的收藏、我的单话点赞和我的评论均按约定分页，已删除评论和下架单话不出现。

## 管理后台：内容与图片运营

**What to build:** 唯一管理员能够登录后台，管理主题标签、漫画系列、漫画单话、运营专题与 COS 图片，并发布或下架单话。

**Blocked by:** 后端：认证与 COS 上传基础；后端：内容与发现 API；后端：读者互动 API。

- [ ] 管理员登录后可维护主题标签、漫画系列、作者署名及运营专题；专题使用独立上传的 COS 封面。
- [ ] 管理员可将四张画格按顺序直传 COS，保存草稿单话，并只在完整时发布。
- [ ] 已发布单话必须先下架才能编辑；发布、下架和删除草稿均得到明确中文反馈。

## 管理后台：评论处置

**What to build:** 管理员能够查看评论及其删除审计信息，并删除任意评论，使其立即从读者端消失。

**Blocked by:** 后端：认证与 COS 上传基础；后端：内容与发现 API；后端：读者互动 API；管理后台：内容与图片运营。

- [ ] 管理员可分页查看评论，并可选择查看已删除评论及其审计字段。
- [ ] 管理员删除任意评论后，读者端评论列表、单话详情和“我的评论”均不再展示该评论。
- [ ] 删除操作保留追溯记录，不物理删除评论数据。

## 真实联调与跨端审查

**What to build:** 用本地或脱敏数据完成管理员、后端和读者端的完整业务闭环验证，并审查跨端契约、权限与风险。

**Blocked by:** 小程序：公开阅读与发现；小程序：登录互动与“我的”；管理后台：内容与图片运营；管理后台：评论处置。

- [ ] 验证“管理员上传并发布 → 读者阅读互动 → 下架不可见 → 重新发布恢复互动 → 系列、专题和月榜读取”的真实 API 闭环。
- [ ] 记录关键请求、响应与页面结果；构建、单测和静态审查不替代真实联调。
- [ ] 完成跨端审查，明确无审核/举报和分享不去重的已知风险未被实现意外扩大。
